# MasaTools ダウンロード数 / 訪問数カウンター

各アプリの **ダウンロード数**（AI Talk Notes・医療用語辞書・VolumeAC・MultiLibLink・WinTimeLock）と、
**自己分析サイトの訪問数** を、管理者だけが見られる Google スプレッドシートに記録します。

- 集計は Google Apps Script + スプレッドシートで完結（無料・追加サービス不要）
- **総数** と **ユニーク（端末ごとの実利用者数）** の両方を自動集計
- 同一端末の重複はユニーク列で除外され、短時間の二重クリックはクライアント側でも抑制
- 数字はスプレッドシートを開くだけで確認（オーナー＝管理者のみアクセス可）

---

## 構成ファイル

| ファイル | 役割 |
|---|---|
| `counter.gs` | スプレッドシートに記録・集計する Apps Script（Google側に貼る） |
| `count.js` | 各ページに読み込ませるクライアント用スクリプト（`/analytics/count.js` で配信） |

---

## セットアップ手順

### 1. スプレッドシートを作る
Google ドライブで新しいスプレッドシートを作成し、URL 内の ID を控える。
`https://docs.google.com/spreadsheets/d/【この部分がID】/edit`

### 2. Apps Script を貼る
そのスプレッドシートで **拡張機能 → Apps Script** を開き、`counter.gs` の内容を全部貼り付ける。
先頭の `SHEET_ID` を、手順1で控えた ID に書き換える。

### 3. 初期化
関数の一覧から **`setup`** を選び ▶ 実行 → 権限を承認。
`events`（生ログ）と `summary`（集計表）シートが自動生成される。

### 4. ウェブアプリとしてデプロイ
**デプロイ → 新しいデプロイ → 種類「ウェブアプリ」**
- 実行するユーザー: **自分**
- アクセスできるユーザー: **全員**

表示された `～/exec` で終わる URL を控える。

### 5. エンドポイントを設定
`analytics/count.js` の先頭 `ENDPOINT` を、手順4の `/exec` URL に書き換えてコミット。
（設定するまではカウントは送信されません＝安全に停止状態）

---

## 数字の見かた

`summary` シートを開くと、アプリごとに次が並びます。

| アプリ | 種別 | 総数 | ユニーク | 最終記録 |
|---|---|---|---|---|
| AI Talk Notes | download | … | … | … |
| 自己分析サイト | visit | … | … | … |

- **総数** … クリック（ダウンロード開始）や訪問の延べ回数
- **ユニーク** … 端末ごとの実数（同じ端末は何回でも1）

生データを見たいときは `events` シート（日時・アプリ・種別・訪問者ID）を参照。

## 計測ポイント

| 計測対象 | キー | トリガー |
|---|---|---|
| AI Talk Notes | `aitalknotes` | ダウンロードページのAPKリンククリック |
| 医療用語辞書 | `medicaldict` | APK / MS-IME / .plist の各ダウンロード |
| Volume Area Controller | `volumeac` | ダウンロードページのAPKリンククリック |
| MultiLibLink | `multiliblink` | Setup(.exe) ダウンロードクリック |
| WinTimeLock | `wintimelock` | Setup(.exe) ダウンロードクリック |
| 自己分析サイト | `iq-test` | ページ訪問（読み込み時に1回） |

新しいダウンロードリンクを増やしたときは、その `<a>` に
`data-mt-download="アプリキー"` を付ければ自動でカウントされます。

---

## プライバシー

- 記録するのは「アプリ名・種別・ランダムな訪問者ID・日時」のみ。
- 訪問者IDは各端末の `localStorage` に保存されるランダム値で、個人を特定しません。
- IPアドレスや氏名などは保存しません。
