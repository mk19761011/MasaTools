MultiLibLink

図書館の貸出状況・予約状況・読書履歴を取得し、表計算ソフトで読みたい本を選んで予約までサポートするWindowsツールです。

最新版 v1.0 をダウンロード 詳しい使い方（ヘルプ） 匿名アンケートに答える

Windows 10 / 11 対応。最新版 v1.0 をこのページからダウンロードできます。

主な機能

一括で情報取得

登録ユーザーの貸出状況・予約状況・読書履歴を図書館サイトから取得し、表計算ソフトで開けるファイルに保存。複数のユーザー（家族）も登録できます。

表計算ソフトで予約

予約候補シートで「予約する」にチェックして実行するだけ。図書館サイトを開いて検索する手間なく、まとめて予約できます。

読書履歴の整理

直近の読書履歴・貸出中・予約中の本を組み合わせ、同じシリーズを自動でまとめて「次に予約する巻」を分かりやすく提示します。

メール監視・バックアップ

図書館からの通知メールを受信したときだけ自動で情報取得（任意）。取得データや設定はデータフォルダに自動バックアップされます。

対応図書館

  • 主に確認しているのは大阪市立図書館の蔵書検索・予約システムです。URLに licsxp-opac を含む LICS-XP / iLis 系の画面構成を前提としています。
  • LibraXX・Calista など他のOPACは画面構成やボタン名が異なる場合があるため、そのまま対応できるとは限りません。
  • 新しい図書館を追加する場合は、図書館設定にログインURLとユーザー情報を登録し、接続確認でログインできるかをご確認ください。

動作報告・情報提供のお願い

現在、動作を確認できているのは大阪市立図書館のみで、それ以外の図書館で正しく動くかどうかはまだ分かっていません。 もしお住まいの地域の図書館で試していただける方がいらっしゃいましたら、ぜひ下記の情報をお寄せいただけると嬉しいです。 いただいた内容は、対応図書館を増やしたり不具合を直したりするために活用させていただきます。

  • 図書館の所在地 — 都道府県・市区町村（例：兵庫県西宮市）
  • 図書館名 — 利用された図書館の名称（例：西宮市立図書館。蔵書検索・予約システムのURLが分かれば、なお助かります）
  • 使用の可否 — 情報取得や予約ができたか、それとも途中でエラーになったか
  • 使用した環境 — Windows のバージョン（10 / 11）など
  • うまくいかなかった場合 — 表示されたエラーメッセージや、止まってしまった操作の状況。あわせてログの送付もお願いできると助かります（画面上に表示された文字をコピーして貼り付けていただければ大丈夫です）
メールで動作報告を送る 件名はそのままで大丈夫です。お名前の記載は不要で、匿名でのご連絡で構いません。分かる範囲のご記入で結構です。

動作環境

  • Windows 10 / 11 を想定しています。
  • 必要なライブラリを同梱したインストーラーとして配布していますので、別途 .NET ランタイムや Python をインストールする必要はありません。
  • 図書館サイトへの接続および情報の取得にはインターネット接続が必要です。
