MultiLibLink ヘルプ

図書館の貸出・予約状況を取得し、次に予約したい本を候補化して、予約操作を支援するツールです。

はじめに 図書館設定 取得と予約候補 予約実行 メール設定 バックアップ 困ったとき

はじめに

まずは図書館設定で利用者カードを登録し、接続確認を行います。接続確認に成功したら、実行画面の 今すぐ取得 で貸出中・予約中・読書履歴をExcelへ保存できます。

1. 図書館設定ログインURLとカード番号、パスワードを登録します。
2. 接続確認登録ユーザーごとに図書館HPへログインできるか確認します。
3. 今すぐ取得現在の貸出・予約状況と読書履歴を取得します。
4. 予約候補を確認Excelの 予約候補 シートで予約したい本にチェックします。
5. 予約するチェックした候補を図書館HPで検索し、予約します。

初めて使う場合は、図書館設定後に 今すぐ取得 だけ実行して、Excelに現在の状態が出ることを確認してください。

図書館設定

  1. 図書館設定 タブを開きます。
  2. 図書館名を選択または追加します。
  3. 図書館ログイン用URLを入力します。
  4. カード番号とパスワードを入力してユーザーを追加します。
  5. 登録ユーザー一覧から対象ユーザーを選び、接続確認 を行います。
  6. 成功したら 設定を保存 を押します。

登録ユーザーは一覧の上から順に予約処理へ使われます。大阪市立図書館では予約上限が1ユーザー15件のため、上限に近いユーザーは自動的に次のユーザーへ切り替えます。

図書館ログインURLが間違っていると接続確認に失敗します。ブラウザで実際にログイン画面が開けるURLを設定してください。

取得と予約候補

今すぐ取得 を押すと、登録ユーザーの貸出状況・予約状況を取得し、データフォルダの 【図書館名】library_data.xlsx に保存します。

同じExcel内に 予約候補 シートが作成されます。これは直近2か月の読書履歴、現在借りている本、現在予約している本を見ながら、次に予約する巻を決めるためのシートです。

項目意味
予約する予約したいシリーズにチェックを入れます。チェックは次回取得後も保持されます。
状態未検索 はまだ図書館HPで検索していない候補です。予約待ちのため保留 は次巻予約を止めている状態です。
検索ワード1 / 2図書館HPで検索する語です。必要に応じて手動で修正できます。
予約冊数同じシリーズの次巻から何冊まとめて予約するかを指定します。

世界史探偵コナンや日本史探偵コナンは、副題ではなくシリーズ名と 歴史まんが を検索語にします。Slam dunk、Band journal book など空白を含む語は引用符付きで検索します。VOLUME 表記のシリーズは VOLUME74 のように巻数しぼり込みを行います。

予約実行

  1. 予約候補を開く でExcelを開きます。
  2. 予約したい行の 予約する にチェックを入れます。
  3. 検索ワードや予約冊数を確認し、Excelを閉じます。
  4. MultiLibLinkで 予約する を押します。
  5. 確認画面に表示された対象を確認して実行します。

予約待ちと取置中の扱い

最終確認巻が予約中で、状態が 待ち の場合は、次巻が先に取置される可能性があるため予約を保留します。最終確認巻が 取置中 または確保済みの場合は、次巻予約の対象になります。

同じ本を重複予約しないための制御

  • 現在の予約シートに同じシリーズ・同じ巻がある場合は予約しません。
  • アプリが予約成立を確認した巻が候補に残っている場合も、再予約しません。
  • 次巻へ進めたい場合は 今すぐ取得 を実行し、予約状況を更新してください。

検索結果が1000件を超える場合は、検索語をもう少し絞ってください。例: 名探偵コナン 少年サンデーコミックス のように、シリーズ名と叢書名を組み合わせます。

メール設定

メール設定は、図書館から届く通知メールの監視や、処理結果の送信に使います。現在の接続テストはGmailのIMAP接続を前提にしています。

  1. メールアドレスを入力します。例: example@gmail.com
  2. Googleアカウントの通常パスワードではなく、アプリパスワードを入力します。
  3. 接続テスト を押します。
  4. 成功したら設定を保存します。

メールアドレスに @gmailcom のような入力ミスがあると接続できません。@gmail.com のようにドットが入っているか確認してください。

バックアップと復旧

MultiLibLinkは、取得結果を保存する前に現在のExcelを自動でバックアップします。通常画面に復元ボタンはありません。誤操作を避けるため、復旧が必要なときだけ以下の手順で手動復元してください。

種類保存場所用途
Excelバックアップ%LOCALAPPDATA%\MultiLibLink\backups\【図書館名】library_data\壊れたExcelや誤更新から戻すための世代ファイルです。
読書履歴ログhistory_backup.jsonlExcelバックアップまで壊れていた場合に、読書履歴だけを復旧するための追記ログです。普段は開く必要はありません。

Excelをバックアップから戻す手順

  1. MultiLibLinkとExcelを閉じます。
  2. データフォルダを開く でデータフォルダを開きます。
  3. 現在の 【図書館名】library_data.xlsx を、念のため 【図書館名】library_data_壊れた可能性あり.xlsx などへ名前変更します。
  4. backups\【図書館名】library_data フォルダから、戻したい日時のExcelバックアップをコピーします。
  5. コピーしたファイル名を 【図書館名】library_data.xlsx に変更します。
  6. MultiLibLinkを起動し、取得データを開く（予約候補入力） で内容を確認します。

history_backup.jsonl は1行が1冊分のUTF-8テキストです。Excelバックアップから戻せない場合に、読書履歴シートへ再投入するための保険として使います。必要になった場合は、元ファイルを消さずに相談してください。

困ったとき

今すぐ取得が押せない

図書館設定で登録ユーザーの接続確認が完了していない可能性があります。登録ユーザー一覧から接続確認を行ってください。

予約候補が思った通りにならない

検索ワード1・2を予約候補シート上で調整してください。副題まで検索語に入れると1冊だけに絞られすぎることがあります。

予約したはずなのに予約状況に出ない

アプリは予約状況一覧への反映を確認できた場合だけ 予約済 と記録します。反映されない場合は 予約未成立（一覧未反映） として、同日中の再実行を止めます。

ヘルプが古い内容に見える

ヘルプボタンを押すたびに、このヘルプをデータフォルダへ上書きして開きます。古い内容が表示される場合は、ブラウザの再読み込みを行ってください。
